## Bug 修复 #6: Component 命名不一致

### 问题
`building.gd` 的 `_parse_costs()` 将建筑定义中的 `"costComponents"` 映射为 `"Component"`(单数)，但游戏中物品实际名称为 `"Components"`(复数)。

### 影响
- `count_items("Component")` 返回 0，但实际有 20 个 Components
- 需要 Components 的建筑永远无法获得材料交付
- `_has_available_materials()` 在匹配时找不到 "Component" 名的物品

### 修复
文件: `scripts/entities/building.gd`

1. `_parse_costs()` (line 76):
   ```
   "Components": cost_list["Component"] → cost_list["Components"]
   ```

2. `get_total_cost_value()` (line 237):
   ```
   "Component": 32.0 → "Components": 32.0
   ```

### 验证
- 修复前: `count_items("Component")=0`, `count_items("Components")=20`
- 修复后: `cost_list` 使用 `"Components"` 匹配物品名
