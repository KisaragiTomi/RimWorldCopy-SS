### 2. AI 需求阈值修复
**问题**: 殖民者 Rest 最低到 0.061 仍不休息，Food 到 0.15 才中断工作

**原因**:
- `job_giver_rest.gd`: `REST_THRESHOLD = 0.2` 太低
- `pawn_manager.gd`: `CRITICAL_REST = 0.1`, `CRITICAL_FOOD = 0.15` 太低
- 殖民者只有在完成当前工作后才重新评估，中断阈值太低导致长时间疲劳工作

**修复**:
| 参数 | 修复前 | 修复后 |
|------|--------|--------|
| REST_THRESHOLD (开始休息) | 0.2 | **0.3** |
| CRITICAL_REST (中断工作去休息) | 0.1 | **0.2** |
| CRITICAL_FOOD (中断工作去吃饭) | 0.15 | **0.2** |

**效果**: Rest < 0.15 从 28 次降至 **0 次**，最低 Rest 从 0.061 升至 0.170

### 3. Harvest/Sow 最大工作距离
**问题**: 殖民者跨越整张地图 (120x120) 去收割/播种远处的植物

**修复**: `job_giver_sow.gd` 新增 `MAX_WORK_DISTANCE = 40.0`，限制最大搜索半径

### 4. Harvest 自动搬运到仓库
**问题**: 收割后产物直接丢在地上，导致 531 个物品堆积，无人搬运

**原因**:
- `job_driver_sow.gd` 的 `_do_harvest()` 只是把收获物 spawn 在收割位置
- Haul 工作优先级（第 15 位）远低于 Harvest（第 11 位），永远轮不到搬运
- 物品堆积导致 ThingManager.things 膨胀至 658+，性能下降

**修复**: 重写 `job_driver_sow.gd`，Harvest 工作流新增搬运阶段：
1. goto_plant → 走到植物
2. harvest → 收割（200 tick）
3. finish_harvest → 摘取产物（不再丢地上，暂存在 driver 中）
4. **goto_stockpile** → 走到最近仓库
5. **drop_harvest** → 放下产物（支持堆叠合并）

无仓库时退回到原始行为（丢在地上）。

**效果**: 地面物品从 531 降至 63，物品不再无限堆积
