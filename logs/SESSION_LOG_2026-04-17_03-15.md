## 已完成的修复

### 1. 渲染系统 Debounce - 解决崩溃 + FPS 双重问题

**问题**: 游戏运行 1-3 分钟后崩溃，无 SCRIPT ERROR，exit code -1。同时 FPS 在 3x 速度下只有 18-29。

**根因分析**:
- `map_viewport.gd` 的 `_on_thing_changed()` 直接调用 `_render_things()`
- `_render_things()` 会 queue_free 所有精灵并重新创建（600+ 个物体）
- 每次 Harvest 触发 4 次信号（删除植物 + 创建物品 + 搬运删除 + 仓库创建）
- 同一帧内频繁 queue_free + new Node 导致引擎级内存崩溃
- 每帧可能触发数次全量重渲染，严重拖累 FPS

**修复**: Debounce 模式
```gdscript
var _things_dirty: bool = false

func _on_thing_changed(_thing: Thing) -> void:
    _things_dirty = true  # 仅设标志

func _process(delta: float) -> void:
    if _things_dirty:
        _things_dirty = false
        _render_things()  # 每帧最多渲染一次
```

**效果**:
| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| FPS @3x | 18-29 | **44-58** |
| 崩溃 | 1-3 分钟 | **14+ 天无崩溃** |
| 稳定性 | 频繁退出 | 长时间稳定运行 |
